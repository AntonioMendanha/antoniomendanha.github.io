Responsividade das imagens

Dando uma limpada no CSS

Escrito por: Antonio Mendanha - 23/07/2020

Já havia notado que a visualização das imagens não estava bacana. O melhor jeito de resolver seria ter um editor de imagens e salvar as imagens já nos tamanhos que vamos trabalhar assim não precisariam de linhas de código no CSS pra ajustar as imagens. Como esse não é o meu caso (não estou utilizando essa Boa Prática ainda), estava com uma linha no CSS colocando as imagens com width: 30%; height: 30%; e toda vez que as páginas eram carregadas em telas pequenas como um Iphone SE por exemplo as imagens ficavam muito pequenas. Substitui a declaração por width: 70vw; height: auto;.
Declarar 70vw (70% da viewport width) significa que a imagem vai ocupar 70% do campo de visão da tela. Essa declaração de valor deixou a visualização das imagens mais bonita, mas algumas imagens se forem muito pequenas podem ficar esticadas e feias com essa configuração. A melhor solução é editar as imagens nos tamanhos corretos como dito acima.

Nos últimos dias também aproveitei para dar uma limpada nos arquivos, a semântica do site estava errada. Somente como uma revisão, as tags <h1> até a <h6> são utilizadas para Títulos e subtítulos sendo <h1> o principal e conforme vai aumentando o número o sub-título vai ficando menor.
O HTML estava com <h4><p></p></h4>. Não faz sentido ter um parágrafo dentro de um sub-título, ou é um sub-título ou é parágrafo, então eliminei o excesso de <h4> do código deixando mais limpo. Ainda existe mais correções desse tipo a se fazer, mas vamos com uma coisa de cada vez. Também dei uma organizada na área café modificando as imagens por imagens gratuitas e modifiquei a seção pra ficar parecida com a área Blog.
