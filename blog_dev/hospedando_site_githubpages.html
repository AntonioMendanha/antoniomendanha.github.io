Hospedando a página no Git Hub Pages

Escrito por: Antonio Mendanha - 13/07/2020

Antes de encarar uma hospedagem paga e uma url personalizada eu optei por verificar a usabilidade do meu site de outra forma, usando o Git Hub Pages. Para quem não conhece o GitHub, o resumo mais rápido possível é: Uma rede social onde cada programador pode mostrar seus projetos e contribuir com outros que estão na plataforma. Eu considero como a minha vitrine atual para poder mostrar a minha evolução e o que venho aprendendo.

Para hospedar um site usando o GitHub Pages, você cria um repositório com o ".github.io" e depois subir os arquivos do seu site para o repositório. Lembrando que você deve seguir as regras de uso do site e também de entender as suas limitações, dentre elas a plataforma só permite páginas estáticas.

Bem, acabei de subir o site e encontrei alguns pequenos problemas. O primeiro é mais importante de todos é: Fiz alguma coisa errada! Não estava conseguindo acessar o site usando o meu link. Fiz algumas pequenas alterações e já consegui acessar. Chegou a hora de compartilhar então o link do meu projeto e pedir opiniões de pessoas próximas. Antes de compartilhar, já alterei o "title" da página Index.html e atualizei a página Blog Dev. Uma das próximas coisas que terei que implantar será a adaptação das páginas para o modelo de blog.

  • Retirar os artigos e criar arquivos separados
  • Criar uma página principal com os link dos artigos
  • Na página principal colocar um Iframe com o conteudo do artigo selecionado

Parece pouca coisa, mas com certeza para um iniciante vai ser um trabalhinho gostoso de se fazer, enquanto também vou colocando aqui as experiências que estou tendo e o que estou aprendendo.

E pra finalizar o artigo de hoje, deixa eu subir as alterações pra você poder ler no site. Um abraço e até o próximo artigo.
