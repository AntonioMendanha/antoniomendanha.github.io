Separando artigos e otimizando metadados

Escrito por: Antonio Mendanha - 18/07/2020

Saiba que se você apaga uma linha errada de um código você pode demorar um tempo até encontrar o erro e corrigi-lo, imagina o que pode acontecer se o seu código estiver gigante. Separar os artigos seria inevitável então vamos logo fazer isso. As etapas então são:

  • Criar os arquivos para os artigos individuais
  • Criar um layout para a página principal da área Blog-Dev com os artigos
  • Incluir os links para os artigos
  • Rodar e verificar usabilidade

A primeira parte até que foi fácil, criar os arquivos individuais para cada artigo foi simples pois foi só copiar o código antigo e apagar algumas linhas deixando somente um artigo por arquivo. Lá dentro do <Head> mudei os títulos (<title>)de cada artigo. Quando o usuário abrir mais de uma aba no navegador vai ficar mais fácil ele saber o que estava aberto já que o título agora é o assunto do artigo aberto. Com relação ao layout da página principal da área Blog-Dev também foi bem simples. Utilizei o esqueleto de um arquivo que eu havia criado antes, uma caixa com uma imagem e título do artigo, inclui a data de postagem pra ficar com a cara de blog e mostrar as atualizações. Nessa estrutura, optei por colocar os mais novos sempre no início. Quando o leitor quer procurar se tem alguma atualização ele sempre procura no início da página.

Com os artigos criados e a página principal pronta foi a hora de criar os links. Como mudei a estrutura de pastas internas tive que prestar atenção nos endereços para funcionar direito. Pronto mais uma etapa concluida, e agora fazer uma limpeza nos olhos arrumando a casa com o CSS. Dei uma primeira organizada nas margens, fixei as imagens no canto esquerdo atribuindo float: left;. Observação: quase esqueci que após atribuir float tem que limpar na próxima declaração pra não quebrar a visualização. E agora sim, mais um item riscado do meu checklist. Ainda tem muita coisa pela frente pra ajustar e mudar, nem cheguei no JavaScript ainda, e quando chegar vai dar um trabalhinho refatorar mas também é inevitável. Ainda preciso corrigr a visualização das imagens, mas isso é um próximo assunto.
