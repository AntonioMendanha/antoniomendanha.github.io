Menu Fixo e Responsivo

Tava feio no mobile

Escrito por: Antonio Mendanha - 17/07/2020

A navegação do site ainda está longe de ser dinâmica, e estou sentindo a necessidade de criar um botão voltar, mas essa inclusão do botão é só após organizar a publicação de artigos e inclusão dos metadados em cada página. Navegando pelo mobile detectei 3 coisas para corrigir (e uma refatorada nos arquivos CSS). A primeira delas é o tópico desse artigo. Melhorar o menu principal.

Optei por fixar o menu porque resolvo o outro problema que é a navegação do site. Com o menu fixo o acesso do usuário as outras partes do site fica mais rápida. Para fixar o menu atribui position: sticky;. Alterei a posição do logo que estava dentro do <aside> joguei ele dentro do menu e atribui um float: left; para fixar ele no início do menu. O logo foi incluido como um <button> pois pretendo deixar ele como um botão pra voltar a Home do site. Dei uma organizada no CSS do menu deixando toda a responsividade do menu no mesmo arquivo. Refatorar o código é importante para facilitar as alterações futuras do código, inclusive inclui alguns comentários dentro do código pois já estava esquecendo o porque tinha atribuido alguns valores e com os comentários vai facilitar a minha vida mais pra frente sempre que quiser alterar algum layout do site.

Apesar de já ter ajustado o menu, a rolagem da página estava me causando desconforto pois as palavras dos artigos estavam sendo misturadas com o menu durante a rolagem. Então atribui um background-color na cor atual do background do site. Agora o outro desconforto era na visualização dos artigos, havia atribuido margens laterais fixas que estavam muito grandes principalmente no mobile. O valor estava atribuido com uma porcentagem muito alta e diminui significativamente deixando o site mais bonito.
Já o terceiro problema detectado eu não consegui parar para vericar hoje, são as imagens que estão nos artigos. Inclusive a partir desse artigo vou colocar algumas imagens das alterações visuais do site a cada artigo para facilitar a compreensão. As imagens estão muito pequenas e não está dando pra ver o que tem nelas a não ser que faça um zoom na mesma, então tenho que ver uma melhor forma de fazer isso.

No próximo artigo vou cuidar dessas correções e começar a usar os metadados, na sequência separar os artigos para otimizar os metadados.
